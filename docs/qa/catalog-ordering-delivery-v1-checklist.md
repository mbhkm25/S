# QA Checklist — Catalog Ordering & Delivery v1

## بوابات الجودة الآلية

- [ ] `npm run lint`
- [ ] `npm run build`
- [ ] `git diff --check`
- [ ] `git status -sb` نظيف ومتزامن

## إعدادات الإدارة

- [ ] تحميل الإعدادات الحالية دون فقدان القيم.
- [ ] حفظ تفعيل وتعطيل قائمة الطلب.
- [ ] حفظ نصوص الأزرار ومقدمة الرسالة.
- [ ] حفظ طريقة عرض السعر والعملة.
- [ ] حفظ شكل البطاقة والمؤثر.
- [ ] اختيار عنصر واحد أو عنصرين مميزين.
- [ ] منع اختيار أكثر من عنصرين.
- [ ] حفظ ترتيب العنصرين.
- [ ] حفظ إعدادات مقدم التوصيل ومناطق الخدمة.
- [ ] حفظ إعدادات الخصوصية ومشاركة الإجمالي.

## الكتالوج العام

- [ ] ظهور العنصرين المميزين بالترتيب المحدد.
- [ ] توقيع صور العناصر المميزة حتى لو لم تكن ضمن أول ستة عناصر.
- [ ] ظهور حالات التوفر بشكل صحيح.
- [ ] عدم ظهور زر الإضافة للعنصر غير المتاح.
- [ ] إخفاء السعر بالكامل عند تعطيل `show_prices`.
- [ ] استمرار فتح صفحة تفاصيل العنصر عند تعطيل الطلب.
- [ ] ظهور رسالة أن قائمة الطلب غير مفعلة دون تعطيل التصفح.

## قائمة الطلب

- [ ] إضافة عنصر جديد.
- [ ] زيادة كمية عنصر موجود بدل تكراره.
- [ ] احترام الحد الأعلى للكمية.
- [ ] تقليل الكمية إلى الصفر يحذف العنصر.
- [ ] حذف العنصر يدويًا.
- [ ] حفظ ملاحظة مستقلة لكل عنصر.
- [ ] بقاء القائمة بعد تحديث الصفحة.
- [ ] فصل قائمة كل نشاط عن الأنشطة الأخرى.
- [ ] حساب الإجمالي منفصلًا لكل عملة.
- [ ] عدم إظهار الإجمالي عند تعطيله.

## المرجع والرسائل

- [ ] ظهور مرجع الطلب داخل القائمة.
- [ ] ثبات المرجع عند تكرار فتح القائمة.
- [ ] استخدام المرجع نفسه في رسالة المتجر ورسالة التوصيل.
- [ ] إنشاء مرجع جديد عند «بدء طلب جديد».
- [ ] استخدام الرابط العام من `VITE_PUBLIC_APP_URL`.
- [ ] عدم مسح القائمة تلقائيًا بعد فتح واتساب.

## بيانات العميل والخصوصية

- [ ] منع إرسال طلب المتجر عند غياب الحقول الإلزامية.
- [ ] منع إرسال التوصيل عند غياب العنوان المطلوب.
- [ ] منع إرسال التوصيل دون موافقة عند تفعيل `require_privacy_consent`.
- [ ] عدم إظهار مربع الموافقة عند تعطيل اشتراطها.
- [ ] إعادة الموافقة إلى غير محددة عند تغيير شركة التوصيل.

## شركات التوصيل

- [ ] تحميل الشركات المنشورة فقط.
- [ ] مطابقة المحافظة والمدينة أو منطقة الخدمة.
- [ ] استبعاد نشاط المتجر نفسه من النتائج.
- [ ] إظهار حالة التوثيق وملاحظة التسعير والتوفر.
- [ ] عدم إظهار الإجمالي في رسالة التوصيل عند تعطيل `share_order_total`.
- [ ] بقاء تفاصيل العناصر وبيانات العميل عند إخفاء الإجمالي.

## المسارات والحالات الاستثنائية

- [ ] فتح `/b/{slug}` بصورة صحيحة.
- [ ] فتح `/b/{slug}/p/{productId}` بصورة صحيحة عند تفعيل الكتالوج.
- [ ] العودة إلى النشاط عند عدم وجود العنصر.
- [ ] عدم انهيار الصفحة عند `slug` أو `productId` غير صالح.
- [ ] إعادة المسار القديم إلى النشاط عند تعطيل الكتالوج عبر متغير البيئة.
- [ ] عمل الروابط داخل المتصفح وCapacitor والإنتاج.
